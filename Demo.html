Is This Already There — Or Does HCL Need to Build It?
Verdict based on AppScan 360° v2.0.0 / v2.1.0 documentation and product research
Already exists in AppScan 360°
  • Org-level centralized dashboard — active issues, MTTR, risk rating, compliance charts across all applications and all scan types
  • Scans & Sessions page — lists DAST, SAST, IAST, SCA scans in a unified list (but navigated separately by category)
  • Correlation engine (IAST-based) — analyzes IAST + DAST + SAST findings to identify common root causes and fix groups
  • RapidFix / GenAI AutoFix — AI-generated fix recommendations with GenAI summaries (SAST only, paid add-on)
  • IFA (Intelligent Findings Analytics) — groups and consolidates hundreds of findings into manageable categories
  • MCP Server — IDE-based LLM triage access across all scan types via Model Context Protocol
  • EPSS scores — real-world exploit probability shown in SCA issue details pane
  • Compliance reports — PDF exports for OWASP, PCI-DSS, NIST standards at portfolio level
Does NOT exist — HCL needs to build this
  • Per-artifact unified view — selecting one project artifact and seeing SAST + DAST + IAST findings side-by-side in a single graphical screen. The Scans page lists scan categories separately; no merged per-artifact view exists.
  • Per-artifact severity donut + OWASP stacked chart + trend — visual breakdown scoped to a single artifact, not the whole portfolio
  • AI Fix Advisor at issue-row level — conversational triage per issue across all scan types (DAST, IAST, SCA), not just SAST via RapidFix
  • Per-artifact compliance gate — pass/warn/fail verdict for one artifact against PCI-DSS, NIST, OWASP with release gate enforcement
  • Role-based artifact views — developer view, CISO view, AppSec view scoped to a single project artifact
  • Cross-scan CWE clustering without IAST — correlation currently requires active IAST subscription; CWE-based grouping across SAST+DAST+SCA without IAST dependency does not exist
  • Security debt score + fix simulation — unified debt metric (Issues × Severity × Age) and pre-fix impact preview across all scan types
Verdict: Partially Exists — The Engine Is There, the Experience Is Not
AppScan 360° has the data, the correlation engine, and the AI fix capability — but they are scattered across separate pages and limited to specific scan types (e.g. RapidFix is SAST-only; correlation requires IAST). What does not exist is a per-project-artifact, unified graphical command center where a developer or security lead selects one artifact and sees every SAST, DAST, and IAST finding merged, ranked, and explained in a single view — with AI fix advice for each issue across all scan types, and a real-time release gate verdict. This is what needs to be developed by HCL.

Implementation Steps — How to Deploy
From zero to live dashboard in 6 steps
Step 1 — Deploy the Dashboard
Open appscan_unified_dashboard.html in Chrome/Edge. No install, no build, no server needed. For team access, host on any static file server (Nginx, S3, Netlify, IIS) or share via corporate intranet.
Step 2 — Connect to AppScan 360°
Enter your AppScan 360° base URL and API key in the dashboard config banner. The dashboard calls GET /api/v4/Issues to pull all SAST/DAST/IAST/SCA issues. v2.1.0 auth: header X-Api-Key: {KeyID}:{KeySecret} — create at Admin → API Keys → Create. 30-min auto-refresh session. Service accounts also supported.
Step 3 — Configure AI Backend
Choose one of three AI providers: Claude API (paste Anthropic API key), Local LLM (install Ollama: curl -fsSL https://ollama.com/install.sh | sh && ollama pull llama3:8b), or OpenAI-compatible (enter endpoint + key). Click "Test" to verify connectivity.
Step 4 — Configure Gate Policies
Define pass/fail rules for Production, Staging, and Development gates. Gate evaluator auto-checks: zero critical issues, no CVSS ≥9.0 in SCA, OWASP compliance, IAST coverage thresholds, license approvals. Override workflow available for break-glass scenarios.
Step 5 — Enrich with EPSS
Dashboard auto-queries api.first.org/data/v1/epss?cve=CVE-XXXX for issues with CVEs across all scan types (SAST/DAST/IAST/SCA) to overlay real-world exploitability scores. Bubble chart and table column display EPSS values for prioritization.
Step 6 — Triage & Remediate
Click any issue row to open the AI Fix Advisor. Choose: Generate Code Fix (before/after diff), Explain Vulnerability (plain English), OWASP Reference, or Business Impact assessment. Export filtered issues as CSV/JSON/PDF.
