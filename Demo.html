Implementation Steps — How to Deploy
From zero to live dashboard in 6 steps
Step 1 — Deploy the Dashboard
Open appscan_unified_dashboard.html in Chrome/Edge. No install, no build, no server needed. For team access, host on any static file server (Nginx, S3, Netlify, IIS) or share via corporate intranet.
Step 2 — Connect to AppScan 360°
Enter your AppScan 360° base URL and API key in the dashboard config banner. The dashboard calls GET /api/v4/Issues to pull all SAST/DAST/IAST/SCA issues. v2.1.0 auth: header X-Api-Key: {KeyID}:{KeySecret} — create at Admin → API Keys → Create. 30-min auto-refresh session. Service accounts also supported.
Step 3 — Configure AI Backend
Choose one of three AI providers: Claude API (paste Anthropic API key), Local LLM (install Ollama: curl -fsSL https://ollama.com/install.sh | sh && ollama pull llama3:8b), or OpenAI-compatible (enter endpoint + key). Click "Test" to verify connectivity.
Step 4 — Configure Gate Policies
Define pass/fail rules for Production, Staging, and Development gates. Gate evaluator auto-checks: zero critical issues, no CVSS ≥9.0 in SCA, OWASP compliance, IAST coverage thresholds, license approvals. Override workflow available for break-glass scenarios.
Step 5 — Enrich with EPSS
Dashboard auto-queries api.first.org/data/v1/epss?cve=CVE-XXXX for issues with CVEs across all scan types (SAST/DAST/IAST/SCA) to overlay real-world exploitability scores. Bubble chart and table column display EPSS values for prioritization.
Step 6 — Triage & Remediate
Click any issue row to open the AI Fix Advisor. Choose: Generate Code Fix (before/after diff), Explain Vulnerability (plain English), OWASP Reference, or Business Impact assessment. Export filtered issues as CSV/JSON/PDF.
